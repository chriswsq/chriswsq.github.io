CasperLabs测试网部署

2021-02-04

主网还未发布
目前只有部署测试网

手动部署


sudo apt remove -y casper-client casper-node

curl -JLO https://bintray.com/casperlabs/casper-debian-tests/download_file?file_path=casper-node_0.7.3-4186_amd64.deb
curl -JLO https://bintray.com/casperlabs/casper-debian-tests/download_file?file_path=casper-client_0.7.3-4186_amd64.deb
sudo apt install -y ./casper-client_0.7.3-4186_amd64.deb ./casper-node_0.7.3-4186_amd64.deb

# Download genesis files
cd /etc/casper
sudo curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/accounts.csv
sudo curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/chainspec.toml
sudo curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/validation.md5
md5sum -c ./validation.md5 


sudo mv /etc/casper/config.toml /etc/casper/config.toml.bak
sudo curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/config-example.toml
sudo ./config_from_example.sh


# Get trusted_hash into config.toml
curl -s 54.183.27.75:8888/status | jq -r .last_added_block_info.hash

# Or do this automatically
sudo sed -i "/trusted_hash =/c\trusted_hash = '$(curl -s 54.183.27.75:8888/status | jq -r .last_added_block_info.hash | tr -d '\n')'" config.toml

sudo logrotate -f /etc/logrotate.d/casper-node
sudo /etc/casper/delete_local_db.sh
sudo systemctl start casper-node
systemctl status casper-node
These instructions will connect you to a small running network.

docker部署

目前casperlabs只支持ubuntu系统，这里采用docker方式部署

Dockerfile

from ubuntu:18.04

RUN apt-get update \
  &&  apt-get install -y --reinstall systemd \
  &&  apt-get install -y curl  \
  &&  apt-get install -y jq  \
  &&  curl -JLO https://bintray.com/casperlabs/casper-debian-tests/download_file?file_path=casper-node_0.7.3-4186_amd64.deb  \
  &&  curl -JLO https://bintray.com/casperlabs/casper-debian-tests/download_file?file_path=casper-client_0.7.3-4186_amd64.deb  \
  &&  apt install -y ./casper-client_0.7.3-4186_amd64.deb ./casper-node_0.7.3-4186_amd64.deb  \
  &&  cd /etc/casper \
  &&  curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/accounts.csv  \
  &&  curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/chainspec.toml  \
  &&  curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/validation.md5  \
  &&  md5sum -c ./validation.md5 \
  &&  mv /etc/casper/config.toml /etc/casper/config.toml.bak \
  &&  curl -JLO https://casper-node.s3.us-east-2.amazonaws.com/release-test/config-example.toml \
  &&  ./config_from_example.sh \
  &&  curl -s 54.183.27.75:8888/status | jq -r .last_added_block_info.hash \
  &&  apt install -y logrotate \
  &&  logrotate -f /etc/logrotate.d/casper-node \
  &&  /etc/casper/delete_local_db.sh
EXPOSE  7777  8888  9999
ENTRYPOINT ["/usr/bin/casper-node","validator","/etc/casper/config.toml"]

docker-compose文件

version: '3'
 
services:

   casper:
     image: casper:0.7.3
     container_name: casper-testnet
     ports:
       - 7777:7777 
       - 8888:8888
       - 9999:9999

     volumes:
       - /bsn/casper/validator_keys:/etc/casper/validator_keys/
       - /bsn/casper/config.toml:/etc/casper/config.toml
       - /bsn/casper/data:/var/lib/casper/casper-node
     restart: always

其中配置文件中的hash值是通过执行命令获取

curl -s 54.183.27.75:8888/status | jq -r .last_added_block_info.hash

known_addresses参数值是找官方社区人员要的

其他命令

获取节点信息

curl http://localhost:8888/status |jq

获取所有rpc节点

curl -X POST -H 'Content-Type: application/json' -d '{"id":1,"jsonrpc":"2.0","method":"rpc.discover"}' 'http://127.0.0.1:7777/rpc' | jq
